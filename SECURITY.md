# Security Policy

## Supported Versions
This is a portfolio/research project. Security fixes are applied to the `main` branch.

## Reporting a Vulnerability
Please report vulnerabilities privately to maintainers first.
Do not disclose publicly until a fix is prepared.

Include:
- Reproduction steps
- Affected files/modules
- Severity assessment
- Proposed remediation (if known)

## Secure Development Guardrails
- Never hardcode credentials or tokens.
- Keep secrets in environment variables.
- Validate data input types and ranges in user-facing interfaces.
- Review fairness and bias implications for model updates.
